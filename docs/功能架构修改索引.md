## 1. 根壳层 / 全局入口

### 根壳层涉及文件

- `AnkiApp/Sources/ContentView.swift`
- `AnkiApp/Sources/DebugView.swift`

### 根壳层修改点

- 导入导出进度遮罩层切到 Theme 语义背景、边框、文字色。
- 顶部用户入口胶囊、同步中状态文字、同步角标统一到 Theme 语义色。
- Debug 页的状态文字、账号信息、列表背景统一到 Theme 背景与次级文本色。

### 影响范围

- 只影响根壳层视觉表现与调试页样式。
- 不改导入导出、切换用户、同步、调试动作等逻辑。

## 2. Review / Image Occlusion

### Review 涉及文件

- `AnkiApp/Sources/Review/CardWebView.swift`
- `anki-upstream` 子模块（已初始化到 `3890e12c9e48c028c3f12aa58cb64bd9f8895e30`，用于上游行为对照）

### Review 修改点

- 对齐上游 reviewer 的 image occlusion 绘制语义，补齐 `occludeInactive` 标记解析。
- 非活跃遮罩仅在 `occludeInactive=1` 时参与 toggle 判定与绘制。
- toggle 重绘路径改为基于可见 shape 重新绘制；隐藏状态下只保留 text shape，恢复时重置 reveal 状态。

### Review 目标问题

- 修复遮罩卡翻面后“切换遮罩”按钮点击无效。
- 对齐上游 Anki 在 reviewer 端的显示 / 切换行为。

### Review 验证边界

- 已做上游实现对照与文件级错误检查。
- 仍需在 iOS 端实际点按 `切换遮罩` 做运行态确认。

## 3. Stats / 数据范围与图表语义

### Stats 数据相关文件

- `AnkiApp/Sources/Stats/PeriodPicker.swift`
- `AnkiApp/Sources/Stats/StatsDualAxisSupport.swift`
- `AnkiApp/Sources/Stats/AddedChart.swift`
- `AnkiApp/Sources/Stats/ReviewsChart.swift`
- `AnkiApp/Sources/Stats/IntervalsChart.swift`
- `AnkiApp/Sources/Stats/HourlyChart.swift`
- `AnkiApp/Sources/Stats/FutureDueChart.swift`
- `AnkiApp/Sources/Stats/ButtonsChart.swift`
- `AnkiApp/Sources/en.lproj/Localizable.strings`
- `AnkiApp/Sources/zh-Hans.lproj/Localizable.strings`
- `AnkiApp/Sources/ja.lproj/Localizable.strings`

### Stats 数据修改点

- 在 `StatsDualAxisSupport` 中新增 `niceUpperBound()`，统一左右 y 轴上界语义。
- `Added`、`Reviews`、`Intervals/Stability`、`Hourly`、`FutureDue` 改为显式固定 `.chartYScale(domain: 0...leftAxisMax)`。
- trailing 右轴补 `AxisTick()`，不再只是“贴标签”，而是更接近独立右侧坐标轴。
- 全局 `RevlogRange` 新增 `month`，顶部范围现在支持“月 / 近一年 / 全部”。
- `ButtonsChart`、`HourlyChart`、`ReviewsChart` 会根据全局范围自动收窄子图 period 选项，避免出现超出全局抓取窗口的假选项。
- 新增 `stats_range_month` 本地化文案。

### Stats 数据目标问题

- 修复统计页右轴曲线与右侧 y 轴刻度不一致。
- 修复顶部只拉一个月数据时，子图仍显示更大 period 选项的问题。

### Stats 数据验证边界

- 已完成文件级错误检查。
- 仍需在 iOS 端确认 `Added`、`Reviews`、`Intervals/Stability`、`Hourly` 的右轴显示与顶部“月”范围切换效果。
