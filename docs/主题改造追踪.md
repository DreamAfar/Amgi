### 2026-04-17 - Phase 1 编译修正（AxisValueLabel 与 Theme 字体）

#### 双轴修正范围

- 根据 CI 日志 `job-logs_2.txt` 修正 Stats 图表轴标签与 Theme 字体扩展的类型兼容问题

#### 本轮已修正文件

- `AnkiApp/Sources/Stats/AddedChart.swift`
- `AnkiApp/Sources/Stats/EaseChart.swift`
- `AnkiApp/Sources/Stats/FutureDueChart.swift`

#### 本轮问题定位

- CI 首个真实失败点出现在 `EaseChart.swift`
- 根因不是工作流路径，也不是依赖缺失，而是 `AxisValueLabel` 不能直接链 `amgiFont(...)`
- 该问题属于 Charts 轴标签类型边界，与普通 SwiftUI `View` 修饰方式不同

#### 本轮修正方式

- 将轴标签字体样式下沉到 `AxisValueLabel` 的自定义 `Text` 内容中
- 保持原有坐标轴文案和统计逻辑不变，只修正类型不兼容的样式写法

#### 功能修复验证结果

- `AddedChart.swift` 文件级错误检查通过
- `EaseChart.swift` 文件级错误检查通过
- `FutureDueChart.swift` 文件级错误检查通过
- Stats 模块内已无 `AxisValueLabel` 直接链 `amgiFont` 的残留匹配

#### 本轮遗留问题 / 下一步

- 继续 Phase 1 后续图表时，轴标签若要套用 Theme 字体，统一使用自定义 `Text` 内容方式

### 2026-04-17 - CI 编译失败排查 AddedChart

#### 错误现象

- GitHub Actions `job-logs.txt` 中首个编译错误出现在 `AnkiApp/Sources/Stats/AddedChart.swift:140:25`
- 错误信息：`contextual closure type '() -> Content' expects 0 arguments, but 1 was used in closure body`

#### 根因

- `AddedChart` 的 `.chartYAxis` 中把右侧 `AxisMarks(position: .trailing, ...)` 错误地嵌套在左侧 `AxisMarks` 的闭包内部
- 在 Xcode 26.3 / Swift 6.2.4 的 Charts builder 下，这一层被推断成无参 content 闭包，导致内部 `AxisMarks { value in ... }` 直接触发编译错误

#### CI 修复结果

- 已将右侧 `AxisMarks` 提升为与左侧 `AxisMarks` 并列的 sibling 结构
- 写法对齐 `FutureDueChart.swift`、`ReviewsChart.swift` 等已通过编译的双轴图实现
- 已额外检查 `FutureDueChart.swift`、`ReviewsChart.swift`、`IntervalsChart.swift`、`HourlyChart.swift` 的同类双轴结构，未发现相同问题
- `AnkiApp/Sources/Stats` 目录级错误检查通过
